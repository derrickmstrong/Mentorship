Cloning a Repository from GitHub

Cloning a repository means creating a local copy of the code and history from a GitHub repository to your computer. Follow these steps to clone a repository:

Step-by-Step Instructions

1. Prerequisites

Before you start, ensure you have the following:

  • Git: Installed on your computer. If not, download and install it from the Git website.
  • GitHub Account: Youâ€™ll need an account to access and clone repositories. Sign up at GitHub if you don't have one.
  • Terminal or Command Line: Access to a terminal or command line interface (CLI) to run Git commands.

2. Find the Repository URL

Follow these steps to get the URL of the repository you want to clone:

  1. Go to the GitHub repository you want to clone in your web browser.
  2. On the repository's main page, locate the Code button (usually green) and click on it.
  4. Copy the repository URL from the dialog that appears. You can choose HTTPS, SSH, or GitHub CLI. For beginners, HTTPS is usually the easiest.
  5. Example URL format (HTTPS): https://github.com/username/repository-name.git

3. Open Terminal or Command Line

Follow these steps to open your terminal and navigate to your desired directory:

  1. Open your terminal (Mac/Linux) or command prompt (Windows). You can use the built-in terminal in your code editor like VSCode as well.
  2. Navigate to the directory where you want to clone the repository using the cd command. For example: 
    3. cd path/to/your/directory

4. Clone the Repository

To clone the repository, follow these steps:

  1. In your terminal, type the git clone command followed by the copied URL. For example: 
    2. git clone https://github.com/username/repository-name.git
  2. Press Enter to execute the command.
  3. Git will start copying the repository to your local machine. You should see output similar to this: 
    4. Cloning into 'repository-name'...
remote: Counting objects: 100% (XXX/XXX), done.
remote: Compressing objects: 100% (XXX/XXX), done.
remote: Total XXX (delta X), reused X (delta X), pack-reused XXX
Receiving objects: 100% (XXX/XXX), X MB | X MB/s, done.
Resolving deltas: 100% (XX/XX), done.

5. Navigate to the Cloned Repository

Once the cloning is complete, navigate into the repository directory using the cd command. Replace repository-name with the actual name of your cloned repository: 

cd repository-name

You can now start working with the code locally!

Troubleshooting Tips

  • Permission Denied (Publickey): If you encounter this error when using SSH, ensure your SSH keys are correctly set up in GitHub.
  • Authentication Failed: For HTTPS cloning, make sure you have the correct username and password or set up a personal access token if 2FA is enabled.
  • No Git Command: If you receive a message that Git is not recognized, confirm that Git is installed and the path is correctly added to your system's environment variables.
